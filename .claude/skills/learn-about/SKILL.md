# 学习研究

研究仓库并创建全面的学习笔记。

## 必需输入

请询问用户：

1. **仓库**（必需）：要研究的仓库名称（必须在 sources.json 中）
2. **主题**（必需）：要研究的方面/模式
3. **类别**（必需）：保存到哪个 docs/learns/ 子目录
4. **相关仓库**（可选）：要比较的其他仓库
5. **优先级**（可选）：P0、P1 或 P2（默认 P2）

## 步骤

### 1. 定位仓库

在 `sources/<类别>/<名称>/` 中查找仓库，如未克隆则运行同步：

```bash
./scripts/sync-sources.sh <名称>
```

### 2. 研究阶段

探索仓库以理解：

1. **目录结构** - 代码如何组织？
2. **核心抽象** - 主要类型/类是什么？
3. **关键模式** - 主题是如何实现的？
4. **代码示例** - 找到代表性代码片段
5. **设计决策** - 为什么这样做？

根据主题关注领域：
- **流式处理**：查找 async/await、生成器、事件发射器
- **错误处理**：查找错误类、重试逻辑、回退
- **上下文管理**：查找历史、内存、上下文类
- **类型安全**：查找类型定义、联合、验证器
- **架构**：查找层分离、模块边界

### 3. 阅读模板

阅读 `docs/templates/learning-note-template.md` 了解结构。

### 4. 编写学习笔记

在 `docs/learns/<类别>/<主题>-<仓库>.md` 创建文件：

```markdown
# [主题] 在 [仓库名称]

> **范围**：[本文档涵盖什么]
>
> **综合自**：[仓库名称]
>
> **优先级**：[P0/P1/P2]

---

## 概述

[2-3 段关于模式/概念]

## [仓库名称] 中的实现

### 核心抽象

[带代码示例的关键类型/类]

### 设计决策

[为什么这样做？]

### 代码示例

[代表性代码片段]

---

## 与其他框架比较

[如果指定了相关仓库，比较方法]

---

## 关键要点

1. [要点 1]
2. [要点 2]
3. [要点 3]

---

## 相关文档

- [相关笔记链接]

---

*创建时间：[今天日期]*
*更新时间：[今天日期]*
```

### 5. 更新索引

运行 update-index 技能或手动更新：

1. 添加条目到 `docs/README.md`
2. 更新 `README.md` 中的计数

### 6. 更新 sources.json

在 `sources.json` 中仓库的 `notes` 数组添加新笔记。

## 示例

用户："学习 pydantic-ai 如何处理流式处理"

操作：
1. 定位 `sources/agent-harness/pydantic-ai/`
2. 研究流式处理实现
3. 阅读 `docs/templates/learning-note-template.md`
4. 编写 `docs/learns/streaming/pydantic-ai-streaming.md`
5. 更新 `docs/README.md`
6. 更新 `sources.json` notes 数组

## 质量标准

学习笔记应该：
- 解释"为什么"而不只是"是什么"
- 包含源码中的实际代码示例
- 如可能至少与一个其他框架比较
- 提供可操作的要点
- 遵循模板结构

## 修改的文件

- `docs/learns/<类别>/<文件名>.md` - 新学习笔记
- `docs/README.md` - 添加到索引
- `README.md` - 更新计数
- `sources.json` - 添加到 notes 数组

## 验证

1. 笔记遵循模板结构
2. 代码示例来自实际源码
3. 索引已更新
4. `sources.json` 包含笔记引用
